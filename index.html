1. Introducción

El suministro de agua es un recurso fundamental y limitado. En distintas instalaciones, tanto domésticas como industriales, las fugas representan un problema constante que ocasiona desperdicio, daños materiales y costos elevados.

El objetivo de este proyecto fue desarrollar un sistema inteligente capaz de detectar fugas de agua en una tubería mediante el uso de sensores físicos, aplicando conceptos vistos en clase tales como:

  • Sensores analógicos
  • Instrumentación acústica
  • Acondicionamiento de señales
  • Medición e instrumentación
  • Interfaz hombre–máquina (HMI)

El sistema fue diseñado para generar alertas tempranas mediante el análisis de presión, vibración y comportamiento dinámico del flujo.

2. Descripción del tema aplicado

Conceptos del curso de Medición e Instrumentación integrados en el proyecto.

Sensores analógicos Instrumentación acústica Acondicionamiento de señal Interfaz HMI

Sensores analógicos: se emplearon para medir variaciones continuas de presión dentro de la tubería, convirtiendo cambios físicos en variaciones de voltaje.

Instrumentación acústica: el sensor acústico registra vibraciones internas generadas por turbulencias o microfugas.

Acondicionamiento de señal: las señales provenientes de los sensores se estabilizan y convierten mediante el ADC del microcontrolador.

Interfaz HMI: la información se presenta en una aplicación web que actúa como panel de monitoreo para el usuario.

3. Desarrollo del proyecto

El proyecto se dividió en tres áreas principales: hardware, método de medición e interpretación de datos.

3.1 Montaje de hardware

1. Sensor analógico de presión 174 psi

  • Rango: 0–174 psi
  • Salida: 0.5 V – 4.5 V (lineal)
  • Propósito: detectar variaciones internas de presión
  • Aplicación: fugas, bloqueos o cambios repentinos en el flujo

2. Sensor acústico industrial

  • Detecta vibraciones internas asociadas a microfugas
  • Capta resonancia de golpes de ariete o turbulencia
  • Permite análisis detallado del comportamiento del sistema

3. Microcontrolador Arduino

  • Recibe señales analógicas desde los sensores
  • Utiliza su ADC (0–1023) para convertir voltaje a valores digitales
  • Envía datos a la aplicación web para su visualización

4. Pantalla integrada

  • Muestra presión y vibración en tiempo real
  • Sirve como instrumento de diagnóstico inmediato en campo

3.2 Método de medición aplicado

El método de medición seguido fue el siguiente:

  1. Instalación de los sensores en puntos estratégicos de la tubería.
  2. Energización del sistema y estabilización del flujo.
  3. Registro de la señal analógica de presión correlacionada con el voltaje.
  4. Captura de vibraciones internas mediante el sensor acústico.
  5. Digitalización de señales mediante el ADC del microcontrolador.
  6. Visualización de valores, gráficos y estado del sistema en la aplicación web.
  7. Clasificación de estados: Normal, Alerta y Posible fuga.

4. Resultados

Durante las pruebas del modelo funcional se obtuvieron los siguientes resultados:

  • El sensor acústico detectó vibraciones anormales incluso ante fugas muy pequeñas.
  • El sensor analógico de presión mostró caídas bruscas al simular fugas en la tubería.
  • La interfaz web mostró los datos en tiempo real de forma clara y ordenada.
  • El sistema reaccionó rápida y eficazmente ante variaciones del flujo.
  • La arquitectura es escalable para agregar inteligencia artificial, actuadores y registros históricos.

5. Conclusiones

Se demostró que la detección temprana de fugas es posible mediante la combinación de sensores analógicos y acústicos. El sistema desarrollado ofrece:

  • Lecturas precisas en tiempo real
  • Monitoreo continuo del comportamiento del flujo
  • Interpretación intuitiva de datos para el usuario
  • Potencial para ampliaciones tecnológicas futuras

Además, el proyecto permitió aplicar de forma práctica conceptos del curso, fortaleciendo los conocimientos sobre medición, sensores, instrumentación y análisis de señales.

7. Anexos: Datasheet de sensores y fórmulas utilizadas

Resumen de especificaciones técnicas y expresiones matemáticas empleadas en el proyecto.

7.1 Datasheet – Sensor analógico de presión (0–174 psi)

Tipo: sensor piezorresistivo industrial

Rango: 0–174 psi

Salida: 0.5 V – 4.5 V (lineal)

Alimentación: 5 V DC

Fórmula del fabricante (conversión de voltaje a presión):

P = ((Vout − 0.5) / 4) × 174

7.2 Datasheet – Sensor acústico industrial

Rango de frecuencia: 35–45 kHz

Salida: señal proporcional a la vibración interna

Aplicación: detección de vibraciones internas por turbulencias o microfugas.

Fórmula de análisis RMS (nivel de vibración):

VRMS = √( (1/N) · Σ(Vi2) )

7.3 Conversión ADC del Arduino

El Arduino utiliza un conversor analógico-digital de 10 bits (0–1023). La relación con el voltaje es:

Vout = (ADC / 1023) × 5 V

7.4 Determinación del estado del sistema

Se definieron criterios de decisión con base en umbrales de presión y vibración:

Parámetro Condición Interpretación
Presión Fuera de ±10% del valor de referencia Anómala · Posible fuga o bloqueo
Vibración RMS Superior al nivel típico de operación Anómala · Microfuga o turbulencia
Presión + Vibración Ambos parámetros anómalos Crítico · Fuga confirmada
